Gerenciamento de Propostas - Documentacao de Uso

Guia rapido para operacao do sistema (admin, propostas publicas, tracking, Clarity e ZapSign).

1) Acesso inicial

  • Painel admin: /admin/login
  • Credencial padrao inicial: admin@local / admin123
  • Recomendacao: trocar usuario/senha apos o primeiro acesso.

Em producao, publique o sistema em HTTPS e restrinja acesso ao painel.

2) Estrutura do painel

Dashboard Visao geral (totais, publicadas, assinadas, views e scroll medio).
Propostas Lista completa, status, links, editar, preview, analytics e duplicar.
Nova proposta Formulario completo com disciplinas, etapas, arquivos e exclusoes.
Configuracoes Clarity, ZapSign, dados da empresa e termos de aceite editaveis.

3) Fluxo recomendado de operacao

  1. Criar proposta em Nova proposta.
  2. Salvar e abrir Preview para revisar layout.
  3. Publicar proposta para gerar token/link publico.
  4. Enviar para assinatura (ZapSign) quando desejar assinatura digital.
  5. Acompanhar leitura, funil e dispositivos em Analytics.

4) Configuracoes importantes

Clarity

  • Ativar script nas propostas publicas.
  • Informar Project ID.
  • Campos de export (endpoint e token) ficam prontos para evolucao de relatorios.

ZapSign

  • Preencher Base URL e API Key.
  • Se o token estiver invalido/expirado, o sistema exibe erro claro no admin e no link publico.
  • Webhook pronto em: /webhooks/zapsign.

Termos de aceite

  • Edite titulo, texto e checkbox no painel.
  • Voce pode usar variaveis como {{PROPOSTA_NUM}}, {{NOME_CLIENTE}}, {{VALOR_TOTAL}}.

5) Regras automaticas do template

  • Revisao 00 nao e exibida nos blocos publicos.
  • Itens de exclusao vazios nao aparecem na proposta.
  • Disciplinas nao selecionadas nao aparecem no escopo/valores.
  • Arquivos recebidos aceitam link e icone de acesso em nova guia.

6) Estrutura de dados e persistencia

ItemDescricao
storage/database.jsonBase principal (admins, configuracoes, propostas, tracking).
storage/backups/Snapshots automaticos do banco JSON a cada escrita.
storage/logs/Logs de execucao local.
public/uploads/Arquivos enviados pelo admin (ex: icones customizados).

7) Publicacao (Apache/Nginx)

  • Apontar Document Root para public/.
  • Garantir PHP 8.1+ com extensao cURL habilitada.
  • Liberar escrita em storage/ e public/uploads/.
  • Definir URL publica em Configuracoes (campo Base URL), quando aplicavel.

8) Checklist rapido de entrega

  1. Login admin funcionando.
  2. Criar/editar/publicar proposta funcionando.
  3. Link publico responsivo e com tracking ativo.
  4. PDF e aceite funcionando.
  5. ZapSign validado com token correto.
